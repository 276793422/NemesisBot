# 桌面自动化 Skill

## 修正说明

⚠️ **重要更正**: 图像分析由 AI 自己完成，不使用任何 MCP 分析功能。

---

## 核心架构

```
用户请求
   ↓
[Skill 控制]
   ↓
┌─────────────────────────────────────┐
│  window-mcp.exe (stdio/stdout)       │
│  路径: Skills/desktop-automation/    │
│         bin/window-mcp.exe           │
│  - 窗口查找                          │
│  - 窗口置前                          │
│  - 坐标获取                          │
│  - 截图（保存为文件）                │
│  - 鼠标点击                          │
└─────────────────────────────────────┘
   ↓ 截图文件
┌─────────────────────────────────────┐
│  AI 分析（自己）                     │
│  - Read 工具读取截图                 │
│  - 视觉识别 UI 元素                  │
│  - 定位元素坐标                      │
│  - 提取文本内容                      │
└─────────────────────────────────────┘
   ↓ 元素坐标
[坐标转换] → 返回 window-mcp 执行点击
```

---

## 三大功能

### 功能 1: 智能截图解析

**流程**:
1. window-mcp: 查找窗口 → 置前 → 获取坐标 → 截图（保存文件）
2. AI: Read(截图文件) → 分析内容 → 返回报告

**用途**: 验证窗口显示、调试布局

---

### 功能 2: 视觉识别点击

**流程**:
1. window-mcp: 查找窗口 → 置前 → 获取坐标 → 截图（保存文件）
2. AI: Read(截图文件) → 识别元素位置 → 返回相对坐标
3. window-mcp: 计算绝对坐标 → 点击

**用途**: 动态布局、未知位置

---

### 功能 3: 代码计算点击

**流程**:
1. window-mcp: 查找窗口 → 置前 → 获取坐标
2. 计算: 根据规律计算元素相对位置
3. window-mcp: 计算绝对坐标 → 点击

**用途**: 固定布局、快速点击

---

## 关键要点

1. ❌ 不要使用 window-mcp 的 analyze 函数
2. ❌ 不要使用任何 MCP 的图像分析工具
3. ✅ 使用 window-mcp 的截图功能保存为文件
4. ✅ 使用 Read 工具读取截图文件
5. ✅ AI 自己视觉分析截图内容
